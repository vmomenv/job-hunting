# 逻辑设计与操作指南 (Logic Design)

本文档旨在为开发人员 (或 AI IDE) 提供如何通过 ADB 和视觉模型操作招聘 App 的详细逻辑流程。

## 1. ADB 操作坐标系
*   **模拟器默认分辨率**：`720x1280 (DPI 320)`。
*   **常用命令**：
    *   `adb shell input tap x y`：在指定坐标点击。
    *   `adb shell input swipe x1 y1 x2 y2 duration`：滑动屏幕 (上滑找岗位)。
    *   `adb shell screencap -p /sdcard/screen.png && adb pull /sdcard/screen.png .`：截取屏幕。

## 2. 视觉识别流程 (Vision Pipeline)
视觉方案负责将原始截图转化为结构化数据：
1.  **目标定位 (Detection)**：寻找岗位卡片 (Job Card)。
2.  **内容提取 (OCR)**：
    *   **岗位名称** (Title)
    *   **薪资范围** (Salary)
    *   **公司名称** (Company)
    *   **职位要求** (Requirements/Skills)
3.  **坐标转换**：将识别出的卡片中心点坐标记录，用于后续点击进入详情页。

## 3. 平台特定逻辑 (App Specifics)

### Boss 直聘
*   **主页**：点击底部的“职位” Tab。
*   **筛选**：点击顶部的“筛选器”，视觉模型需识别并点击对应的层级。
*   **详情页**：识别岗位描述 (JD) 全文，LLM 分析后，如果匹配，点击右下角的“立即沟通”。

### 智联招聘 / 猎聘
*   逻辑基本一致。关键在于识别每个 App 不同的 UI 图标和布局差异。

## 4. 简历分析逻辑 (Resume Matcher)
这是核心的智能决策部分：
1.  **Prompt 模板**：
    ```text
    任务：基于求职者的简历，分析以下岗位描述 (JD) 的匹配度。
    求职者简历：内容...
    目标岗位描述：内容...
    请返回：
    1. 匹配度分数 (0-100)
    2. 匹配的核心理由 (3条)
    3. 不匹配的主要冲突项 (1条)
    ```
2.  **决策流**：
    *   `Score > 80`: 自动点击“沟通/投递”。
    *   `50 < Score < 80`: 记录到 Excel 待人工复核。
    *   `Score < 50`: 直接跳过。
