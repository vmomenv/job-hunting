# 逻辑设计与操作指南 (Logic Design)

本文档描述了如何通过视觉模型指导控制真实安卓设备，实现智能求职。

## 1. 核心业务流程 (Core Workflow)
1.  **初始化**：读取 `config.yaml` 中的期望岗位（如“Python 开发”）和最低薪资待遇。
2.  **视觉巡航**：
    - 自动打开招聘 App (Boss/智联/猎聘)。
    - 视觉模型识别输入框，定位坐标，输入关键词搜索。
    - 识别搜索结果页中的“过滤器”，设置薪资等要求。
3.  **智能筛选**：
    - 对搜索结果列表进行截图，发送给视觉大模型 (VLM)。
    - VLM 返回当前可见岗位的名称、公司、薪资及点击坐标。
    - 脚本根据配置比对薪资，若符合则点击进入详情。
4.  **详情深度评估**：
    - 截取岗位详情页。
    - 视觉模型提取 JD 全文。
    - 判定匹配度，若合适则记录到 Excel。

## 2. 视觉操作指令 (Vision Actions)
不同于传统的 OCR，本项目采用 **VLM (Vision-Language Model)** 产生决策：
*   **Prompt 演示**：
    > "当前屏幕是一个招聘软件的搜索页面。请找到所有标题包含 'Python' 且月薪不低于 '15k' 的岗位，返回它们的中心坐标和公司名称。"
*   **动作执行**：
    - 获取 VLM 返回的坐标 `(x, y)`。
    - 调用 `adb shell input tap x y`。

## 3. 简历分析逻辑 (Resume Matcher)
1.  **简历向量化** (可选)：提取简历关键标签 (技能、经验)。
2.  **匹配评分**：
    - `Score > 85`: 记录到 `jobs_table.xlsx` 并标记为“极力推荐”。
    - `Score > 60`: 记录到 Excel 待参考。
